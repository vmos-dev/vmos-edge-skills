# Install Third-Party Apps

只在任务明确是“安装、下载并打开某个第三方应用”时读取这一页。

## 适用场景

- 用户要求安装某个应用，例如微信、抖音、闲鱼、Telegram、WhatsApp
- 用户要求“下载安装并打开”某个应用
- 用户要求打开某个第三方应用，但当前不知道它是否已安装，或直接启动失败

## 完成标准

- 目标应用已安装
- 能成功启动到前台
- 已通过当前可用观察接口完成验证

## 推荐流程

### 1. 先检查是否已经安装

- 如果用户已经明确给了 `package_name`：
  - 先试 `/activity/launch_app`
  - 如果能启动成功，就直接返回“已安装且可正常打开”
- 如果还不知道包名：
  - 先查 `/package/list?type=user`
  - 能按应用名称精确匹配到目标包名时，再试 `/activity/launch_app`
- 如果直接启动失败、包名未找到，或列表里没有目标应用，再进入下载流程
- 不要优先改成手动翻桌面、文件夹或应用抽屉找图标

### 2. 如果用户已经给了 APK 链接或文件

- 用户直接给了下载链接：
  - 优先 `/package/install_uri_sync`
- 用户已经把 APK 文件放到当前环境可读位置：
  - 优先 `/package/install_sync`
- 安装后继续第 5 步做验证

### 3. 如果只有应用名，先用浏览器搜索下载页

- 国内应用可优先搜索品牌官网或主流下载站
- 国外应用可优先搜索官网 APK 下载页
- 使用 `/activity/start_activity` 打开浏览器搜索页或下载页
- 浏览器包名优先级：
  - 先看 `/package/list` 里是否有 `mark.via`
  - 没有再尝试 `com.android.chrome`
- 搜索结果与下载页观察优先级：
  - 网页主要看文本、链接和按钮时优先 `/accessibility/dump_compact`
  - 要确认视觉布局、下载弹窗、按钮相对位置时再补截图

### 4. 下载后安装

- 如果页面能直接拿到稳定下载链接，优先 `/package/install_uri_sync`
- 如果必须先在浏览器里下载 APK：
  - 处理“下载”“仍要下载”“确定”等确认弹窗
  - 等待下载完成后，再安装本地 APK
- 只有在没有专用接口可用时，才考虑用 `/system/shell` 辅助定位最新下载文件

### 5. 验证安装结果

- 优先使用安装接口返回的 `package_name`
- 如果返回里没有可用包名，再查 `/package/list?type=user`
- 启动验证优先级：
  - 首启授权场景：`/activity/launch_app`
  - 普通启动：`/activity/start`
- 用 `/activity/top_activity`、`/package/list`、截图或 `dump_compact` 验证应用确实已打开

## 重要约束

- 应用名称要精确匹配，避免把“微信”装成“微信读书”，或把“抖音”装成“抖音极速版”
- 用户已经给了明确安装源时，不要额外兜圈去网页搜索
- 下载页主要是 WebView 文本任务，默认先用 `dump_compact`，不是默认只靠截图
- 删除旧版本、卸载其他同类应用、清除应用数据前先确认用户意图
