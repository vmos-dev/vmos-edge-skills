# Browser Search And Reading

只在任务明确要求在云机浏览器里搜索、打开网页、提取页面内容时读取这一页。

## 适用场景

- 用户要求在云机里搜索某个问题、新闻、百科、天气、股票或网页信息
- 用户要求打开某个网页并读取其中内容
- 用户要求在设备浏览器里完成搜索，而不是只要模型自己联网回答

## 何时使用

- 如果用户强调“打开浏览器”“在云机里搜”“在手机页面里看结果”，使用这页
- 如果只是普通信息查询，且不要求必须在设备浏览器里完成，优先使用 agent 自身可用的联网能力，不必绕设备浏览器

## 浏览器选择

- 默认优先 `mark.via`
- 如果 `/package/list` 里没有 `mark.via`，再尝试 `com.android.chrome`
- 不要默认依赖“系统默认浏览器”

## 推荐流程

### 1. 构造搜索词或目标 URL

- 去掉“帮我查”“搜一下”这类无关词，只保留核心关键词
- 天气类可改成 `<城市名> 天气`
- 新闻类可改成 `<话题> 最新消息`
- 百科或说明类优先主题词本身

### 2. 用浏览器打开页面

- 用 `/activity/start_activity`
- 常见参数组合：
  - `package_name`: `mark.via`，没有再用 `com.android.chrome`
  - `action`: `android.intent.action.VIEW`
  - `data`: 搜索 URL 或目标网页 URL
- 搜索页可优先用 Bing 这类结果页，或用户指定的站点

### 3. 先观察页面文本

- 页面主要看标题、摘要、段落、链接文本时，优先 `/accessibility/dump_compact`
- 如果存在登录、通知、权限等弹窗，先关闭再继续提取
- 如果 `dump_compact` 已经足够回答问题，就不要额外做重截图

### 4. 需要更多信息时再进入详情页

- 根据 `dump_compact` 选择最相关结果
- 再用 `/accessibility/node`、`/input/click` 或其他当前更合适的交互接口进入详情页
- 页面内容不够时，再用滚动接口继续观察

### 5. 输出结果

- 先整理出用户真正需要的结论
- 如果任务要求“只观察不操作”，不要额外点击更多页面
- 如果任务要求把当前网页状态继续交给后续动作，再明确当前前台页面和可点击区域

## 重要约束

- 设备内网页任务默认先看文本，不要一上来只靠截图
- 只有在需要确认视觉布局、图片、遮挡、按钮位置时，才补截图
- 浏览器场景默认优先 Via，不要无原因切到别的浏览器
